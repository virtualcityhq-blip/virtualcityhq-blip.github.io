INTERNAL PUBLISHING TOOL

Sendlines Shorts Publisher.

A private, owner-operated integration that uses the YouTube Data API to publish approved Sendlines videos to YouTube @sendlinespress.

What it does.

The tool verifies the authorized Sendlines channel, uploads one approved vertical video with its exact approved title and description, and reads the resulting video status back from the YouTube Data API. It records success only after the destination, metadata, and visibility are verified.

Who can use it.

This is an internal company tool used only by authorized Virtual City LLC personnel for the single Sendlines-owned channel. It is not a public software service, does not accept third-party accounts, and does not publish on behalf of customers.

Permissions.

The OAuth consent flow requests only youtube.upload and youtube.readonly. The tool does not request YouTube Analytics, advertising, comments, subscriptions, messaging, or payment permissions.

Safety controls.

Every upload is approval-bound to an exact video and exact channel-specific copy. The publisher checks schedule and cadence, verifies the expected channel ID, prevents duplicates, never blindly retries an ambiguous upload, and stops if authorization or public-visibility verification fails.

Required operator consent.

This public page is descriptive and does not accept files, start OAuth, or upload to YouTube. In the owner-operated command-line client, a read-only preflight displays the current privacy-policy version, exact title and description, channel, audience declaration, exact approved post hash, selected MP4 SHA-256, and selected YouTube visibility. At commit, the operator must choose either Confirm or Return to edit. Confirm requires interactive policy acceptance, YouTube's upload certification, an exact Made-for-Kids selection, visibility selection, and one exact, one-shot confirmation. Return to edit creates a new draft revision, preserves the superseded item's evidence, clears the new revision's approvals, routes it to independent Quality and Legal review, and stops the upload. A changed file or reviewed field invalidates consent and stops before contacting YouTube.

Authorization health.

Interactive acceptance writes a non-secret receipt for the current published policy version. Only while that receipt remains current may the unattended health check refresh the stored OAuth grant and verify that it still resolves to the expected Sendlines channel. Through governed state mutation, the check records its readiness status, timestamp, channel-match result, dependency category, current policy version, and an explicit marker that no credential value was stored; it never writes credentials or provider-returned identifiers. Missing credentials, provider failures, and network failures remain distinct. A typed invalid_grant is treated as authorization that cannot be reconfirmed, causing cleanup and a publication freeze rather than being mislabeled as proof of user revocation.

Data controls.

Credentials remain in the operating system's encrypted credential store. See the privacy policy for collection, retention, deletion, and revocation details. Access can be revoked through Google Security Settings.

Policies.

Use of this tool is subject to the Sendlines integration terms, the YouTube Terms of Service, the YouTube API Services Terms of Service, and the YouTube API Services Developer Policies.
